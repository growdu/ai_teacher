# 架构设计文档

## 1. 项目概述

### 1.1 项目名称
**AI Teacher Studio** - 智能教学资源生成平台

### 1.2 项目定位
面向教育行业，将教学知识点快速转换为教学视频和教学PPT的智能生成平台。

### 1.3 核心场景
```
知识点输入 → 智能生成 → 教学视频 + 教学PPT
```

## 3. 核心模块设计

### 3.1 知识理解引擎
**职责**：解析输入的教学知识点，提取关键概念、生成课程结构

**能力**：
- 知识点解析与结构化
- 课程大纲自动生成
- 教学重难点识别

### 3.2 内容生成引擎
**职责**：生成教学文案、讲解脚本

**能力**：
- 讲稿生成（文字脚本）
- 教案生成（教学设计）
- 试题生成（随堂练习）

### 3.3 媒体合成引擎
**职责**：将内容合成为最终媒体产物

**能力**：
- PPT生成
- 视频合成（数字人口播）
- 音频合成（TTS配音）

## 4. 生成流程设计

### 4.1 核心流程
```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  知识点输入  │───▶│  课程规划   │───▶│   讲稿生成  │
└─────────────┘    └─────────────┘    └─────────────┘
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    ▼                     ▼                     ▼
            ┌─────────────┐        ┌─────────────┐        ┌─────────────┐
            │   PPT生成   │        │  数字人口播  │        │   试题生成   │
            └─────────────┘        └─────────────┘        └─────────────┘
                    │                     │                     │
                    └──────────┬──────────┴──────────┬──────────┘
                               ▼                      ▼
                       ┌─────────────┐        ┌─────────────┐
                       │  教学PPT    │        │  教学视频   │
                       └─────────────┘        └─────────────┘
```

### 4.2 两个核心输出
| 产物 | 说明 |
|------|------|
| 教学PPT | 包含课程结构、讲解要点、图表的演示文稿 |
| 教学视频 | 数字人口播讲解，配合PPT画面的合成视频 |

## 5. 数据模型设计

### 5.1 知识点表
```sql
CREATE TABLE knowledge_point (
    id bigint PRIMARY KEY,
    tenant_id bigint,
    subject varchar(64),
    grade varchar(32),
    content text,
    created_at timestamp
);
```

### 5.2 课程表
```sql
CREATE TABLE course (
    id bigint PRIMARY KEY,
    tenant_id bigint,
    knowledge_point_id bigint,
    title varchar(255),
    outline jsonb,        -- 课程大纲结构
    script text,          -- 讲稿脚本
    status varchar(32),
    created_at timestamp
);
```

### 5.3 生成的PPT/视频表
```sql
CREATE TABLE teaching_material (
    id bigint PRIMARY KEY,
    tenant_id bigint,
    course_id bigint,
    material_type varchar(32),  -- ppt/video
    file_url varchar(512),
    thumbnail_url varchar(512),
    duration int,               -- 视频时长(秒)
    status varchar(32),
    created_at timestamp
);
```

## 6. 多租户设计

### 6.1 租户模型
```
Tenant (机构)
  └── Workspace (工作空间)
         ├── 知识点库
         ├── 课程
         ├── 生成的PPT
         └── 生成的视频
```

### 6.2 角色
| 角色 | 权限 |
|------|------|
| 管理员 | 机构管理、额度配置 |
| 教师 | 创建课程、生成资源 |
| 制作人员 | 视频制作、后期处理 |
| 审核员 | 审核生成的资源 |

## 7. 技术架构要点

### 7.1 核心设计原则
- **一键生成**：输入知识点，自动完成从大纲到视频的全流程
- **模块化解耦**：知识理解、内容生成、媒体合成独立模块
- **厂商无关**：通过抽象层接入各类AI厂商
- **异步处理**：视频生成等长任务异步执行

### 7.2 可扩展方向
- 题库生成
- 微课生成
- 教案生成
- 试卷生成
